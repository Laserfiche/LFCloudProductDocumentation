Batch Importing Of User Accounts With a Comma Separated Values(CSV) File

Administrators can predefine a list of users in a comma separated values (CSV) file for import into .

On the Users page, click Import from CSV file. The import process supports the following format:

Column Name Additional Notes
EmailAddress*  
FirstName*  
LastName*  
Name* Specify a unique username. A username can contain the following characters and is limited to a maximum length of 47:
  • a-z (case preserving but case-insensitive)
  • 0-9
  • ' (single quote)
  • _ (underscore)
  • - (hyphen)
  • . (period)
Password* This value becomes optional if you select the Send a password setup email option in the Import Users dialog box.
LicenseType Specify a license to reserve for the account. Valid values are:
  • Full
  • Community
  • Participant
  • None

Full allows the user access to all applications.

Community allows the user read-only access to Documents and the ability to participate in Forms processes, but without the ability to authenticate with .

Participant allows the user read-only access to Documents and the ability to participate in Forms processes, plus the ability to authenticate with .

None allows the user access to , but no access to any other service.

Attributes with a * are required.

Sample CSV file:

EmailAddress,FirstName,LastName,Name,Password,LicenseType
johnsmith@Company.com,John,Smith,johnsmith,asdf1234,Full
janesmith@Company.com,Jane,Smith,janesmith,qwerty1234,None
