Configuring and Using the Integration

To use the integration, you will first need to add an STP Alias and associate it with a form. You can then configure properties for the forms that will be stored in . Once this has been done, users can use the integration to store forms to .

Adding an STP Alias

  1. Browse to the Manager Login page and log in.
  2. Click the Administration tab.
  3. Click STP - Aliases.
  4. On the Straight Through Processing Configurations page, click Create New.
  5. In the STP Alias Configuration dialog box:
    1. In Alias/Configuration Name, specify a label to identify this STP alias.
    2. In Button Text, specify the desired toolbar button label. When you open a form, the form tab contains this toolbar button.
    3. In Posting Format, specify XML w/ PDF or XML w/ fillable PDF.
    4. In Set Test Mode, specify the desired option.
    5. In Destination URL, select the URL to your web service. The URL should look similar to the following, with 000ab000 replaced with the actual value in your URL:

      6. https://r-000ab000.laserfiche.com/LaserAppReceiver/ReceivePDFLAAnywhere

      Note: Your administrator must contact to add your URL to the Destination URL drop-down list.

  6. Click OK.

Associating an STP Alias with a Form

  1. Browse to the Manager Login page and log in.
  2. On the Dashboard tab, click Search for Forms.
  3. In the Filter by drop-down list, select desired form and click the ADD TO QUEUE icon.
  4. Under the Open Forms button, click Configure STP Alias.
  5. In the Associate Forms with STP Alias dialog box, select the STP Alias you created above and click Save.

Note: Use the Edit Bindings button on the Administration tab to associate multiple forms with an STP Alias at once. You can also use the Edit Bindings page to remove forms from an STP Alias.

Configuring Integration Settings

The Integrations section of Repository Administration allows administrators to configure the way your forms will be stored in . These settings should be configured before users store forms in . You can set field mapping, the target folder, and the document name. Administrators can also use tokens to set these properties dynamically.

  1. If you are not already in the repository administration section of , click your user name in the upper right corner and select Repository Administration.
  2. Under Additional Items, select Integrations.
  3. Select the Laser App tab.

    Note: The following properties can be configured dynamically using tokens that correspond to information in . For a list of available tokens, see Anywhere Tokens.

  4. In the Field mapping option, configure the way that will store information about the form in the document's fields:
    1. Click Add/remove fields and select the field in which you want to store information from your form. Click OK.
    2. Click the token button (>) to select a token that corresponds to the information you want to store, or type the token name.
  5. In the Target folder option, type the path where you want to store forms. To set the path dynamically, click the token button and select a token that corresponds to the information you want to store, or type the token name.
  6. In the Document Name option, type the name of the saved forms. To set the name dynamically, click the token button and select a token that corresponds to the information you want to store, or type the token name.
  7. Click Save at the top of the page to save your configuration changes.

Using the Integration to Store Forms in

Once an administrator has configured the integration settings, users can download the integration to store forms in . The integration must be downloaded by ever user who will use the integration, and is downloaded in the repository.

  1. In , go to the Dashboard.
  2. Under Filter by, select your form.
  3. Select Associate an STP Alias with a Form.
  4. Click the post button.
