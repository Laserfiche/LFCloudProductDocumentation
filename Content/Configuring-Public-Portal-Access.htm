Configuring Access to Public Portal

Your Laserfiche installation comes with a special account named PublicPortalUser. Everyone who accesses your repository via Public Portal will automatically be signed in as PublicPortalUser. This means that you can set security for the user, and those settings will be used by everyone who accesses your repository through the Public Portal page.

It is important to set security carefully for the public portal user. For example, if the public should only be able to see the contents of one folder, you should make sure that the PublicPortalUser has Browse and Read rights for that folder, but does not have rights for any other folder in your repository. You can also use security to restrict them in other ways: if you do not grant the PublicPortalUser the Print/Export privilege, they will not be able to download files from your repository or print them. See the Security chapter of the Laserfiche Cloud help files for more information.

By default, PublicPortalUser does not have access to your repository. This allows you to configure your security before giving public users access to the repository. Once you have configured security for PublicPortalUser, open the Account Administration page, select Users, select the PublicPortalUser user, and then set Access to Repository to Allow.

Note: Once you set the Access to Repository to Allow, the PublicPortalUser will be a part of the Everyone group. By default, the Everyone group is set with full entry access rights on the root folder of the repository. Ensure you set explicit entry access rights, feature rights, and privileges for the PublicPortalUser, otherwise the PublicPortalUser will inherit all rights of the Everyone group.

View Public Portal

To view the public portal, click the username at the top right, then click View Public Portal.

The URL used to access the portal public is:

https://portal.laserfiche.com/Portal/browse.aspx?repo=r-repositoryID, where repositoryID is the ID number of the repository.
