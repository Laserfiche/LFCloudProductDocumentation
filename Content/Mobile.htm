Capturing Documents with Laserfiche Snapshot

Laserfiche Snapshot is a virtual printer that allows you to "print" electronic documents into your repository as image pages. In effect, Laserfiche Snapshot works as though you had printed the document and then scanned it back into Laserfiche, but allows you to skip the step of making a physical printed copy. This allows you to create documents with pages that you can review and annotate within Laserfiche.

Note: When capturing a document with Snapshot, only the pages will be retained, not the original file itself.  If you want to keep the file, you can import it instead.

Snapshot can capture any electronic file that supports printing in its native application. For each Snapshot capture, you can configure the paper size, color format, and other properties used to generate pages. You can also set metadata for the documents when you capture them. In addition, Snapshot automatically generates text from your documents, which means they will be text-searchable in the repository.

Once you have installed Snapshot on a particular computer, it will automatically be listed among your printers in the Print dialog of applications on that computer.

Printing to Your Repository with Laserfiche Snapshot

  1. Open the document you want to capture in its native application.
  2. Open the application's print dialog.
  3. In the list of printers, select Laserfiche Snapshot. Configure print options in your application and then begin printing.
  4. Laserfiche Snapshot will launch and prompt you to provide your Laserfiche server, repository, user name, and password.
  5. In the Document Properties tab, specify the document name in the Name option. By default, this will be the file name of the document you are printing.
  6. In the Save to Repository option, specify the destination folder in the Folder option. You can use the token button (>) to dynamically set this based on document properties, and the browse button (...) to browse to a location.
  7. In the Template option, you can modify the template and field information for your new document. See Fields for more information.
  8. In the Tags tab, you can add tags to the document. See Tags for more information.
  9. In the Document Handling tab, you can specify how to handle name conflicts when creating the document.
  10. In the Text tab, you can determine whether to generate searchable text directly from the file, or to perform an OCR process to generate text. In general, we recommend selecting Obtain text from the print job, as this usually produces the most accurate results.
  11. Click OK to print the document to the repository.
