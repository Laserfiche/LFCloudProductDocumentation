Outflow

Outflow settings let you specify when a particular pathway in your process should run. For example, after a user task in which a manager approves or denies a form, outflow settings let you designate one course of action for an approved form and another course for a denied form.

Once a gateway is connected to other objects in the process model, its outflow settings will appear in its settings window. Tasks also have outflow settings in the Outflow tab of its settings window.

  • A task or gateway's Outflow settings are shown in a table that lists each destination object the task or gateway is connected to.

    • In the Name column, you can edit the name of the sequence flow (the line that connects the object to a following object).
    • In the Path Conditions column, you can specify conditions that must be met for that particular outflow path to be taken. When you click in the field, the Conditional Expression Builder will open, which you can use to create an expression. Or, if you know XPath, you can enter conditions directly into the text box.
    • To set an outflow path as the default, select the appropriate This is the default outflow path checkbox.
