Creating a Process Diagram

In the Process Modeler, you can build and configure the selected process using objects.

Tip: Before you begin, be sure you understand the business process well enough to translate it into the appropriate objects. Much of the difficulty in designing a business process comes from a lack of knowledge about how the process works or about how to use BPMN.

Three types of objects are used to represent business processes:

  • Events
  • Activities
  • Gateways

The palette on the left in the Process Modeler shows all available objects for building your process, broken down into these three categories. Click the links above for more information about each object type. Once you're familiar with these objects, follow the steps below to learn how to build a business process with them.

At the most basic level, a business process must have a start event and an end event. For this reason, these objects are present on the canvas by default for all processes.

Note: Sub-processes do not need a start or end event.

Creating a Process in Laserfiche Forms

  1. Sign in to Laserfiche and use the application picker or the Overview page and select Forms.
  2. Click Manage.
  3. Click New process () and select from one of the basic templates to create a new process.
  4. Ensure your process has a start event. If one is not present on the canvas, drag one of the Start Event types onto the canvas.
  5. Drag an object from the palette to add it to the canvas. Connect one object to another using Sequence Flows.
  6. Repeat the previous step with various objects until your business process is fully represented on the canvas with objects correctly connected to each other.

    7. Tip: To quickly add an object that needs to be similar to an existing object, right-click the object and select copy. Make any necessary changes to the new object.

  7. Configure events, activities, and gateways used in your process by double-clicking them and opening their settings.
  8. As you build out your process, you can zoom and pan the canvas. Additionally, you can view the process as an image by clicking the View Process Image () toolbar button.
  9. When you are finished configuring your business process, click the validate () toolbar button to check for any errors in your process. If present, errors will be displayed in a pop-up warning message.
  10. Resolve any errors, and click Save to save the business process.

Note: After building a process in the Process Designer, you must publish it before the process can be used.
