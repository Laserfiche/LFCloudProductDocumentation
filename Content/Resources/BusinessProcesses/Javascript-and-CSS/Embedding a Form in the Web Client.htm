Embedding a Form in the Web Client

A form can be embedded in a web client custom tab. This allows the user to look up multiple documents while filling in the form, or to do lookups or calculations on the form while browsing the repository.

Adding a Custom Tab

To add a custom tab in the web client in version 10.3.1 or later, you must use an account that is a member of at least one of the following groups on the web client server computer: The Administrators group or the Web Configuration Managers group.

The conditions for adding a custom tab in the web client in versions 10.3 and earlier are the same as above, except that the account has to be a Windows account, not a Laserfiche user. You do not have to sign in from the web client server computer to add custom tabsâ€”you can configure custom tabs remotely as long as you meet the above conditions. The Admin user account is not able to add custom tabs in these earlier versions of the web client.

To add a form as a custom tab

  1. Sign in to the web client as a user who meets the conditions described above.
  2. Click on your username in the top right corner, then click Options.
  3. Click Display. Navigate to the Tabs section of the Display page.
  4. Click Add next to Tabs. This brings up the Custom Tab dialog.
  5. Enter the Tab name and the URL of the form you want to display in the tab. You can use the token picker () to include variables in the URL, in case your form URL is related to any of the available variables.
  6. Under Location, select the locations that you want the custom tab to be visible in.
  7. Under Display, select the tab's display locations and audience.
  8. Click OK.

Passing Credentials to Forms

When you access a form through a custom tab, you are first signed in to Laserfiche through the web client. If the form is not publicly accessible, the Forms server will need to authenticate your identity before allowing you to view the form in the custom tab. Unless you have Laserfiche Directory Server single sign-on configured, you will need to re-enter your credentials to view a non-public form.

To configure single sign-on for viewing embedded forms

  1. Ensure that the web client server and the Forms server are licensed by the same Directory Server.
  2. Open the web client configuration page and navigate to the Services page. Make sure that the toggle next to Directory Server is turned to ON. If it is turned to ON, there will be a input box for entering the Directory Server STS URL. Make sure that this URL uses the fully qualified domain name of your STS instance.

  3. On your Forms server configuration page, navigate to the User Authentication page and make sure that you are using Laserfiche Directory Server for authentication. In the Server field for this authentication method, use the fully qualified domain name of your Directory Server.
