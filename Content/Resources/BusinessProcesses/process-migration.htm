Process Migration

Processes can be moved between the Global context and a team project if needed to control access based on your team structure. Projects let you organize access to resources and limit access to data collected by those resources.

To migrate a process from the Global context into a team project:

  1. Locate and select the process to be moved within the business processes - manage page.
    (Add a simple screenshot showing Manage header and Migrate button.)
  2. Click the Migrate button in the toolbar above the process list (icon image).
  3. A dialog will open. To cancel migration, simply close the dialog before the final Confirm button.
  4. Step 1: Team and Project:
    1. Select the team from the dropdown or type the team name to search for the team.
    2. Select the project from the dropdown or type the project name to search.
    3. Click Next.
  5. Step 2: Resources:
    1. If there are any other process automation resources linked to this process (lookup tables for instance), they will be listed here. By default, if the security settings for the selected team does not already allow access to Global resources, these resources will be added to the Project Access to Global Resources area of the team settings.
      1. To prevent one or more of the listed resources from being added to the project security settings, uncheck the box next to its name in the list.

        Note: If you deselect or otherwise deny access to any of the resources listed in this dialog, the process will be suspended any time it attempts to access that resource. Once the process has been moved, you must create a resource to replace any missing resources within the project or otherwise revise the process to avoid this issue.

    2. Click Next.
  6. Step 3: Summary
    1. This page provides information about the migration before the process is moved.
    2. Optional: If you select the Once migration is complete, go to the project name management page. option, once complete the page will redirect to the process Manage page within the project where the process will now be listed. Otherwise, the main Manage page will be displayed.
    3. Click Confirm to complete the migration.

To migrate a process from a team project into the Global context:

  1. Navigate to the process Manage page within the project.
  2. Locate and select the process to migrate (screenshot)
  3. Click the Migrate button.
  4. A dialog will display any project related resources. To cancel migration, simply close the dialog before clicking Confirm.

    Note: Any resources located within the project will not be available to the process once it has migrated to global and the process will be suspended any time it attempts to access a listed resource. Once the process has been moved, you must create a global resource to replace any missing resources or otherwise revise the process to avoid this issue.

  5. Optional: If you select the Once migration is complete, go to the Global management page. option, once complete the page will redirect to the main process Manage page, otherwise the project Manage page will be displayed.
  6. Click Confirm to complete the migration.
