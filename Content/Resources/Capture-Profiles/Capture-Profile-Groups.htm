Capture Profile Groups

Capture Profile Groups let you create a group of profiles that the Run Capture Profile activity can use to choose the best matched profile in order to identify and capture accurate information from a document.

Example: Sunshine Daycare is processing old and new admission forms for their child care program. The forms contain the same information, but the layout has been updated and information has been moved to different locations on the new form. In this scenario, you could create a capture profile to retrieve the information on the old form, and another capture profile to retrieve the information on the new form. Then, create a capture profile group and add these two profiles to the group. When the Run Capture Profile Workflow activity runs, it will compare each form scanned to the profiles in the group, identify which form matches which profile, and extract the correct information from each one.

To create a Capture Profile Group

  1. On the Capture Profiles page, click Capture Profile Groups in the left pane. You can also click the New button or click the drop-down and select Capture Profile Group.
  2. Give the Capture Profile Group a name and optional description, then click Create.
  3. On the Capture Profile Groups page, select profiles you want in the group from the Available Profiles list on the left. Use the Search bar at the top to search for a profile.
    • Double-click the profile to add it to the group.
    • Select the profile and click the right-facing arrow.
    • To add multiple profiles at once, press Ctrl, select them, and click the right-facing arrow.
  4. The added profiles will be displayed in the Profiles in Group section, along with the profile version. If changes are made to a profile, you will have the option to update the profile listed in this group by clicking Update.

    If changes are made to a profile:

    1. Save the profile changes.
    2. Click the Refresh button to refresh the profiles in the group.
    3. Click Update next to the updated profile.
    4. Click the Publish button to republish the profile group.
    5. In Workflow, update the version of the capture profile group used in the Run Capture Profile activity.

    Note: If a profile added to a profile group has no anchored zones, it will not be selected to run by the profile group.

  5. Clicking on a profile in the Profiles in Group section will display a preview of the first page of the sample document in that profile. You can click Open profile to view and make changes to the profile.
  6. Under All Profile Zones on the right, you will see a list of every zone created by every profile in the Profiles in Group list. Since only one profile is ultimately run, zones not provided by every profile may not have a value. To see the profile a zone is missing from, hover over the Not captured by # profiles link next to the zone.

    Example: Typically, profile groups are used when documents contain the same information located in different places, maybe due to a document format update. Sometimes, profiles added to a profile group contain different or additional zones to be captured. In this case, if the profile chosen to run contains zones that do not exist in the other profile, those zones will return empty.

  7. To remove a profile from a group, select it in the Profiles in Group section and click the left-facing arrow.
  8. When finished creating the group, you can save it, then publish it.

Note: Capture Profile Groups replaced the Auto-classification feature previously configured in the Run Capture Profile activity.

Note: Smart Invoice profiles are not supported when using Capture Profile Groups.

Note: The more anchored zones configured per capture profile, the more accurate identification and capture will be. At least one anchored zone must be configured per capture profile in order for it to work properly.
