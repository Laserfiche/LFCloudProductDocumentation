Configuring Entry Access Rights

Process automation uses a system account “ProcessAutomationUser” to connect to your Laserfiche Cloud repository. The system account manages process automation access to entries in the repository. If the "Everyone" security group is denied access to an entry, for example, this system account can still be granted access to that entry and process automation can still make changes in the repository. See Security to learn more about how to configure the entry access rights for the system account that manages process automation access to entries in the repository.

Important: You must set the entry access rights for the system account in your Laserfiche Cloud repository.

Workflow Activities

Workflow is able to make changes in your Laserfiche Cloud repository using the ProcessAutomationUser system account. This system account must have specific access rights enabled in order for activities to make changes to the entry or entries within a folder within the repository. This section details the minimum entry access rights that you must grant the system account in order to use each activity. If you do not want to allow a specific type of action to be performed, be sure to disable the respective access right.

Note: See List of Access Rights to learn more about each entry access right described below.

Note: Activities not included in the list below do not require any entry access rights to the repository.
