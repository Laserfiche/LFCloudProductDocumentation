Troubleshooting Workers

You can troubleshoot and view the status of your workers by navigating to the worker in the Workers tab of a remote agent. These logs can help you determine why a worker stopped, when you need to reconfigure a worker, or when a worker is not performing as expected.

The logs contain the following sections:

  • Worker Status: This section states the last time the worker made a successful call to Process Automation.
  • Status: This section says whether the worker is enabled or disabled.
  • Events: This section provides a detailed log of when major events occur for the selected worker (e.g., when there is a connection error, when a resource is deleted, when authorization fails, etc.).
  • History: This section describes the progress of the worker's work items. For example, if the work item involves looking up a data source, it can indicate whether the data query successfully ran.

Viewing the Version of a Remote Agent Worker

  1. From the configuration page for your remote agent, click the Workers tab. For each registered worker take note of the displayed version number.

Viewing the Event Log for a Remote Agent Worker

  1. From the configuration page for your remote agent, click the Workers tab. For a specific registered worker, expand the Events section, to view a log of when major events occur for the selected worker.

Common Error Events

View the Events of a registered worker to see a log of major events.

Event Type Notes
Authorization failure

The worker cannot connect to Laserfiche Cloud.

One common cause for this error type is if the worker version is outdated.

If the worker version is outdated:

  1. Remove the outdated worker.
  2. Download the latest version of worker installation package.
  3. Reinstall the worker on the appropriate machine and re-register the worker.
