Importing and Exporting Rules

The importing and exporting features make it easy to store rules offline and share rules with others. All rules are imported and exported in .bri files.

To import a rule:

  1. Select Import on the top left of the Rules page.
  2. Click Choose File.
  3. Select the .bri file you want to import.
  4. Click Next.
  5. In the next few steps, specify how the imported data will interact with existing data, clicking Next at the end of each step.
  6. On the last Import Summary step, check that you've selected the correct options, then click Save.

To export a rule:

  • Select any rule(s) on the Rules page and click Download in the top right corner of the pane.
  • Double-click a rule in the list of rules to open the Rules Editor, then click Export in the toolbar.

  • Open the Version History and Usage dialog and click Export on the right.

Importing Old Versions of Entity Queries

Entity queries that were exported prior to the April 2019 release of Laserfiche Cloud have a different format from entity queries exported after that release. The following happens when you import an entity query in the old format to Laserfiche Cloud:

  • Laserfiche will attempt to match the old rule format to the new one as far as possible. Parts of your query may be unfilled or look different in some cases. You will be asked to verify the updated query as displayed in the rule editor.
  • When you publish the rule in its new format for the first time, you will overwrite the old query data. You will be warned of this when you first attempt to publish the new rule.
  • Publishing the new rule counts as a major version change because of the change in format.

 

 

 

 

 

 

 

 

You can import and export rules as .bri files. The .bri file type contains one or more rules. Once a rule is exported in a .bri file, it can be shared with others and stored offline. You can also import and export rules by importing and exporting workflows that contain the rules.

To import a rule:

  1. Select Import on the top left of the Rules page.
  2. Click Choose File.
  3. Select the .bri file you want to import, click Next.
  4. Specify how you want to handle existing data.
  5. Click Import.

To export a rule:

  • Select any rules on the Rules page, select Export on the top right, and enter a name.
  • Double-click a rule in the list of rules, select Export in the toolbar, and enter a name.
  • Open the Version History and Usage dialog, select Export on the right, and enter a name.
