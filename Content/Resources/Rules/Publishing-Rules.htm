Publishing Rules

The publishing feature in Rules allows you to make a rule ready to run in processes.

It is a useful way to ensure that both the rule creator and process designer are using a rule that will not break the processes to which the rule is added. Publishing indicates to the process designer that the rule is not going to break any processes.

After publishing more than one rule, you want to ensure that the process designer knows the correct version to use in workflows and business processes. You can ensure they are using the correct version by referencing the version number. The version number updates each time a rule is published. Learn more about versioning.

Testing Rules

Before you send a rule over to the process designer, it is a good idea to test that the rule works as you expect. Each rule editor has a test feature in the toolbar that allows you to enter values for each input to the rule. After you run the test, an output is produced, which you can compare to your expected result.

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

Publishing rules makes the rules ready to run in a workflow. It also updates the rule status from Draft to Published in the right pane, and increments the minor version number. Workflow runs the latest major published version of a rule.

To publish a rule:

  1. Make changes to the rule.

    Note: You can revert to the last-saved version of a rule when you make unwanted changes.

  2. Save the changes.

    Note: For a decision table, you should also validate that the saved changes do not contain configuration errors.

  3. Publish the changes.

You should also test-run the rule in the rule editor to ensure that the rule runs as you expect.

To test-run a rule:

  1. Click Test in the toolbar.
  2. Provide a value for each input.
  3. Click Test.
