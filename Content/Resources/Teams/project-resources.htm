Project Resources

Project resources are the process automation elements that users and processes will have access to within a project. These currently include: business processes, decision tables, formulas, queries, surveys, and lookup tables, and custom reports. Any resources created within the project can be used throughout that project, but are not available to other projects or to other Global resources.

In addition to the project resources, you can allow a project to access resources from the Global project. You may allow all global resources for a specific type to be accessed and used or only specific resources if you do not want a project to have access to everything within Global. Resources that can be shared from Global to a project include decision tables, formulas, lookup query rules, workflows and more.

Note: Project-specific resources are not available to other projects.

To add resources to the project

  1. Navigate to the project landing page.
  2. Open the designer for the type of resource. See the links above for details on each resource type.
  3. Follow the instructions for the specific resource type to create or import the resource.

Migrating Global Resources to a Project

Administrators can export a global resource and re-import the resource into a Team project.

Be aware that this creates a duplicate resource. For example, business processes migrated in this manner will not have their instance data preserved and administrators will need to remap or configure other process automation resources that the business process may interact with.

See Process Migration for instructions on moving business processes between the global project and team projects.

Laserfiche is actively working on integrated methods to migrate other existing resources from the global context to a project.
