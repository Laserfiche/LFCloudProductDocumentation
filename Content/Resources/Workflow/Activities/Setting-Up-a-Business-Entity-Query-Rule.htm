Configuring an Entity Query Rule

This configuration of the Run Rule activity allows you to reference an entity query, perform the custom query on various inputs in the process, and produce some result. You can then use the results of the query elsewhere in your workflow process.

To configure an entity query rule:

  1. Drag the Run Rule activity from the toolbox pane and drop it in the Designer pane.
  3. In the Rule property box, select Entity Query Rule.
  4. Select the rule and version. The version corresponds to the "Major" version number for the rule.
  5. Under Input Values, specify the token and/or hard-coded values you want to use for each input parameter in the WHERE clause.

    Example: Because the Name member uses the input parameter %(name), you can define the parameter in the properties pane. In the following example, the input parameter has the token value %(Name).

  6. Under Output Values, specify the name and tag of the token that contains the result of the rule from each statement in the SELECT clause.

    Example: The following output parameters Teacher_Name and Teacher_Type are referenced in the properties pane in the Workflow Designer.

  7. Under Rows to Return, specify the number of outputs to return.
