Basic Terminology

Below is a list of common workflow terms that you should be familiar with. For general business process definitions, see Business Process Terminology.

  • Laserfiche entry: A document, folder, or shortcut stored in a Laserfiche repository.
  • Workflow: A repeatable pattern of activities arranged in the workflow designer that will interact with entries in a Laserfiche repository, data, or outside sources. A workflow must be published and assigned a starting event before it becomes operational. View a sample workflow.


  • Activity: A single, pre-built, configurable step in a workflow that interacts with Laserfiche entries, data, or outside sources. Activities are the building blocks of workflows. More information. Show me some activities.

  • Publishing: The process of validating, saving, and making the workflow active. More information.
  • Starting Events: A schedule or condition a workflow uses that defines when it will run. More information.
  • Rules: A set of instructions for how to perform actions. Used by all Process Automation features. More information.
  • Workflow instance: A particular running, terminated, or completed workflow. More information.

Note: See the Laserfiche User Guide if you are unfamiliar with basic Laserfiche terminology.
