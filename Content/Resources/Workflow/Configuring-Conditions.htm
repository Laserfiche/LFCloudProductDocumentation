Configuring Conditions

The condition editor has nested sets of conditions that let you specify the conditions that must be met for a workflow, activity, or branch of an activity to run. The editor has one root group that dictates the number of conditions that must be true or false. Within the group are the conditions. You can use the condition editor in several workflow activities--Wait for Entry Change, Repeat, Conditional Decision, Conditional Parallel, Conditional Sequence--and starting events.

To configure a set of conditions

  1. Click No condition set. Click here to create one.
  2. Click the group to specify if any or all conditions in the group must be true or false.

  3. Optional: Add groups by clicking the Add group button. Remove groups by clicking the down arrow, then Delete.
  4. Click the condition type for an activity (first image) or starting event (second image) to specify to what type of entry, user, or token the condition pertains. Point to items in the menu to see more options for specifying the entries or tokens. Note that only the condition types appropriate for the current activity or event will be available. For example, you cannot specify token conditions in starting event conditions, since tokens only exist after a workflow has started. Show me a list of the options

  5. Click the starting entry to specify the entry on which you want to have the condition evaluated. You can only configure this option for the following activities: Conditional Decision, Conditional Parallel, Conditional Sequence, and Repeat.

  6. Click the operator to define the relationship between the left and right items. The operators available in this list depend on the condition type text.

  7. Click the condition value to specify a value for the condition. To add a token, click the right arrow .
