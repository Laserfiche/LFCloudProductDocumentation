Viewing an Instance Report

You can interpret the progress and performance of your workflow instances using the instance reporting feature in the Instances tab. This report is available on the Monitor and Design pages. From the navigation bar on the Monitor page, you can easily find any instance report using the nearby search feature, while from the navigation bar on the Design page, you can easily navigate between the associated workflow definition and the instance report.

Interpreting for an Instance Report

The various tabs in the instance reporting feature let you interpret the progress and performance of a workflow instance. The following table describes when you would use each tab.

  • Workflow:  View the workflow design, activity statuses, activity properties, and description for each activity. Allows users to easily see which activities in the workflow succeeded or failed, and which paths within conditional branches were processed.

  • Details: View general statistics about the instance's performance. Provides insight into how long the workflow took to complete, showing possible bottlenecks in the process or areas where revision for optimization might be necessary or desired.

  • Entries:  View the entries used, along with each entry's folder path and repository. Provides for verification that processes are acting on the right entries, or verification that desired entries are not missed in processing.

  • Activities: View a list of each activity, activity duration, workflow duration, status, and more. Provides a more detailed view of each activity's performance than the Details section allowing for a more granular view.

  • Messages: View a list of the errors, warnings, and informational messages for an instance.

  • Parameters: View a list of the input and output parameters and their values within an instance.

  • Tokens: View a list of the instance tokens along with the current value of the token. Allows users to review the tokens as they flow through the workflow.

Click the Open Instance button next to the instance in the left pane to open the report from the Design to the Monitor page.

Click Recent Activity on the Design page to view the latest workflow activity on the Monitor page..

Viewing an Instance Report

The instance report provides a detailed summary of a workflow instance.

Workflow

This tab displays the workflow design, with each activity's current status. Colored borders surround each activity, indicating the status of the activity. Use the key on the top right to learn the meaning of each color. The two buttons on the top left, Show Summaries and Show Descriptions, let you include details and the description for each activity in the workflow. When you select an activity, the pane on the right displays the activity properties.

Details

This tab provides general statistics and information about the instance. The first section, Summary Charts, provides various graphs that let you compare the performance of the instance to other instances of the same workflow. The second section, Workflow Details, lets you view the workflow name, the starting event that started the workflow, the version number, the start time, and more. Click View to open the workflow in the workflow designer; click the Search button to search for other workflow instances by the same initiator.

Entries

This tab lists entries the workflow interacted with, and the folder path and repository associated with each entry.

Activities

This tab contains a list of the activities the workflow performed, along with the activity duration, workflow duration (the time the activity actively completed a task, as opposed to waiting), status, and more.

Messages

This tab contains a list of the errors, warning, and informational messages for the instance. Errors are unexpected events that prevent the instance from continuing. Warnings are unexpected events that do not prevent the instance from continuing. Informational messages are events that produce additional information when an instance runs.

The Messages tab lists errors, warnings, and informational messages about the workflow instance.

  • Error: An unexpected event that permanently prevents the workflow from continuing.

    Example: A Move Entry activity is configured to move an expense report into the Accounting folder. If the expense report is deleted before the Move Entry activity runs, the workflow will end with an error.

  • Warning: An unexpected event that does not prevent a workflow from continuing.

    Example: A workflow contains an Add Sticky Note activity. When a folder goes through the workflow instead of a document, the following warning appears in the Messages tab of the workflow instance details: Annotations can only be added to documents. The entry is a folder. The workflow will continue even though a sticky note was not added to the entry.

  • Informational Message: An event that produces additional information when a workflow instance runs.

    Example: The Find Entries activity returns all the entries in a specified folder. The informational message for this activity will list how many entries the activity found.

Parameters

This tab displays the unique identifier of the instance as type Input.

Tokens

This tab lists the tokens in the instance, with the current values of the tokens. Tokens appear in this tab when you include a Track Tokens activity or run a workflow from the designer.

 

 

 

 

 

Viewing an Instance Report

The workflow instance opens in a pane in the Monitor section that may contain the following tabs:

  • Workflow: Displays the workflow design with each activity's current status.
  • Details: Provides general statistics and information.
  • Entries: Lists which entries the workflow has interacted with.
  • Activities: Lists the activities that have run and their associated statistics.
  • Messages: Lists errors, warnings, or informational messages.
  • Parameters: Displays the unique identifier of the workflow instance.
  • Tokens: Lists the current values of tokens in the workflow.

Note: You can go back to the original workflow or rerun the original workflow by clicking the More Actions button on the report tab.

Workflow

The Workflow tab displays the workflow design. Different colored borders appear around the activities indicating the activities' statuses or if they encountered errors or warnings. The colors correspond to the colors in the key on the top right.

  • Select an activity to view its properties in the pane on the right.
  • Select Show Summaries to include details about the performance of the activity within each activity box in the instance overview.
  • Select Show Descriptions to include the description from the Description text box in the properties pane.

Details

The Details tab provides general statistics and information about your workflow instance.

  • Summary Charts: Use the Instance Duration graph to see how long the instance took to run compared to the workflow's average runtime.
  • Workflow Details: View the workflow's name, the starting rule that kicked off the workflow, the workflow's version number, the times it started, the time it ended or was last modified (if the workflow has not completed yet), its overall duration, and the instance ID.
    • Click View next to Workflow to open the workflow in the Design page.
    • Select the Search button to search for other workflow instances started by the same user.
  • Starting Entry: View the starting entry's name, folder path, server, repository, and ID.

Entries

The Entries tab lists entries the workflow interacted with and the folder path and repository associated with each entry. The information shown in this tab varies depending on the rights you have.

Activities

The Activities tab lists activities the workflow has performed with their duration, work duration (the time they have been actively completing a task as opposed to waiting), status, start time, and end time.

  • Use the Previous button and Next button to view the activities when you have more than 25 results.
  • Use the Columns button to remove headers from the table.

Messages

The Messages tab lists errors, warnings, and informational messages about the workflow instance.

  • Error: An unexpected event that permanently prevents the workflow from continuing.

    Example: A Move Entry activity is configured to move an expense report into the Accounting folder. If the expense report is deleted before the Move Entry activity runs, the workflow will end with an error.

  • Warning: An unexpected event that does not prevent a workflow from continuing.

    Example: A workflow contains an Add Sticky Note activity. When a folder goes through the workflow instead of a document, the following warning appears in the Messages tab of the workflow instance details: Annotations can only be added to documents. The entry is a folder. The workflow will continue even though a sticky note was not added to the entry.

  • Informational Message: An event that produces additional information when a workflow instance runs.

    Example: The Find Entries activity returns all the entries in a specified folder. The informational message for this activity will list how many entries the activity found.

Parameters

The Parameters tab displays the unique identifier of the workflow instance as type Input.

Tokens

The Tokens tab lists the tokens in the workflow instance with the current values of the tokens. Tokens appear in this tab when you include a Track Tokens activity or run a workflow from the designer.
