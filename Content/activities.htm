Activities

There are two kinds of activities in : tasks and sub-processes. Tasks are activities performed by a user (user tasks) or an application (service tasks). Sub-processes are processes that occur within the larger business process you are creating. Like the processes they are a part of, sub-processes contain their own start and end events.

User Tasks

User tasks represent actions by participants in a business process. For example, a manager approving a form submission or a user revising a previously submitted form would be represented with user tasks.

Why use a user task?

Use a user task if someone needs to take action in the business process. If a user should be notified but will not take action, use an email service task. Public users cannot participate in user tasks.

Configuring general options

Select how users can interact with this task.

  1. To make the form read-only for users who are completing this task, select the Make this form read-only for users this task is assigned to check box.
  2. To allow users to perform this task using the details pane, select Allow direct approval.
  3. The Assign tab will assign the task to the users or team you specify. To allow users and team managers to reassign the task after it's been automatically assigned, select Allow the task to be reassigned. If this option is selected, users will have a Reassign option that lets them assign the task to another user or team. If this option is cleared, users can only assign the task to themselves but not to anyone else.
  4. Select the Automatically load the next task if the same person is assigned to it option if the next user task in the process is assigned to the same individual as this task and if you want the next user task's form to appear immediately after the user submits this user task's form.

    5. Note: This option will only automatically load tasks if the tasks are assigned to the same user. It will not automatically load tasks if the tasks are assigned to a group of users or team, even if the user completing the task is in both groups or teams.

  5. Under Action Buttons, select the check-box next to each button you want visible to the user performing this task. In the field next to each button, you can change its text.
  6. Under Due Date, specify when this task must be completed by. Configuring due dates gives users a way to organize and prioritize tasks and provides helpful reporting metrics for managers. You can define an exact due date, a relative due date based on a variable in your process, or no due date at all.
  7. Under Priority, configure conditions that can change the task's priority. A task's priority appears in the My Tasks section and helps call attention to the task.

Assigning the Task

Tasks can be automatically assigned to users or a team. Tasks can also be reassigned during the process to allow for more flexibility.

  1. In the Assign tab, select if you want to assign the task to one or more users by selecting Users. Alternatively, select Team to assign the task to a team.
  2. If you selected Users, type one or more user's names. Click the variable button to assign the task based on a field value or process variable.
  3. If you selected Teams, chose a team from the drop-down menu. You can also create teams from this menu by selecting Create team.
    1. Optional: You can select or create a filter that assigns the task to specific members of the team based on certain criteria. Learn more about filters.
    2. Under Task Distribution, select if the task will go to the team in general (and appear in the team member's Available tab) or if the task will be automatically assigned to a single team member following a round-robin pattern. If you select Round robin, and the task has never been assigned to a team member before, a team member will be randomly assigned the task. Future tasks will go to the user who has been assigned this user task (either automatically or manually reassigned the task) least recently. If the task is assigned to a user and they release the task, it will go to the team's Available tab, it will not be reassigned round-robin.
  4. Select the check-box next to Email users when this task is assigned to email users assigned to this task when this task is first assigned to them.

    5. Note: Users will not receive this notification if the task is manually reassigned to them. They will only receive this notification when the process runs this user task.

  5. To include field or process information in the subject or body of the email message, click the Insert Variablebutton.
  6. Click Done.

Service Tasks

Service tasks represent actions by a service or program in a business process. For example, sending an email notification to users involved in the process is a service task because it is performed by .

Why use a service task?

Use a service task for sending an email notification or saving a form to your repository. If a user must take action as part of the task, use a user task instead.

Email service task

Email service tasks automatically send an email to one or more users at a specified point in the business process. These service tasks are useful if a user does not have to take action in a process but should still be kept informed of its progress.

To send an email during a process

  1. Place a Email Service Task onto the canvas and connect it to the objects that precede and follow it in the process.
  2. Double-click the task to view its Properties.
  3. Fill in the From, Send to, Subject, and Body fields. Click the Insert Variable button to insert a placeholder for process or field information such as the name of the person who started the process, the submitter, any user task actions or comments, and field values.

Save to Repository service task

To save a form to a repository during a process

  1. Place a Save to Repository Service Task onto the canvas and connect it to the objects that precede and follow it in the process.
  2. Double-click the task to view its Properties.
  3. In the Repository drop-down list, select your repository.
  4. Select the form data you want to save to the repository.
    1. In the Document name field, type a name for the form submission document in the repository. Click the Insert Variable button to insert the value from form or process variables into the name.
    2. In the Path field, enter the path in the repository where the document will be created. Click the Browse... button to browse to a path.
    3. Under Save Options, select to save the document as a PDF or TIF. Alternatively, select No form to save a document that has metadata but no actual image or electronic document component. If you clear this option, the folder path you specified above will be created. (You can apply metadata to this folder with the Parent folder fields option below.) You may want to clear this option if you're only using a form to gather the metadata that you will apply to a folder where the form's attachments will be saved.
    4. Select Include action history to include the action history from preceding user tasks on the form. Select Save business process data as XML if you would like to save an XML copy of the business process with the document.
    5. Under Fields, click Configure fields to select a template and fields to apply to the document. In the Configure fields dialog box, choose a template from the drop-down menu and/or click Add/remove fields to choose the fields you want applied to the form. You can specify a value for each template field and any added fields. Click the Insert Variable button to populate the field with form or process variables. Click Done when you have finished configuring the fields.
    6. Under Parent folder template, click Configure fields to select a template and fields to apply to the folder the form submission document is saved in. In the Configure fields dialog box, choose a template from the drop-down menu and/or click Add/remove fields to choose the fields you want applied to the form. You can specify a value for each template field and any added fields. Click the Insert Variable button to populate the field with form or process variables. Click Done when you have finished configuring the fields.
  5. If your form has file uploads, a tab will appear for each upload. Configure these tabs as follows:
    1. In the Document name field, enter a name for the file upload document in the repository. Click the Insert variable button to insert the value from form or process variables into the name.
    2. Select whether the file upload document should be saved in the Parent form location or Other location. If you select Other location, click the Browse... button to browse to a path.
    3. In the Document relationships field, choose the relationship you want to apply between the uploaded file and the submitted form.
    4. Under Fields, click Configure fields to select a template and fields to apply to the uploaded file. In the Configure fields dialog box, choose a template from the drop-down menu and/or click Add/remove fields to choose the fields you want applied to the uploaded file. You can specify a value for each template field and any added fields. Click the Insert Variable button to populate the field with form or process variables. Click Done when you have finished configuring the fields.
  6. When you have finished configuring the Save to Repository service task, click Done to return to the process diagram. Be sure to click Save to save your changes.

Configuring Outflow

Outflow settings let you specify when a particular pathway in your process should run. For example, after a user task in which a manager approves or denies a form, outflow settings let you designate one course of action for an approved form and another course for a denied form.
