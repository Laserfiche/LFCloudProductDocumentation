Introduction

The Laserfiche API development team is using Swagger UI to allow you to interact with the API through a visual playground. Test out the Laserfiche API by visiting one of the following links depending on where your Laserfiche Cloud repository resides:

  • For the United States Datacenter: https://api.laserfiche.com/repository/swagger/index.html
  • For the Canadian Datacenter: https://api.laserfiche.ca/repository/swagger/index.html

How to first authenticate in the Swagger UI sandbox

  1. In the AccessToken section, expand the Post /v1-alpha/Repositories/{repoId}/AccessToken/Create endpoint
  2. Click Try it out.
  3. Within the repoId, type in the repository ID for your Laserfiche Cloud repository in the format: r-xxxxxxxx

    4. Note: The repository ID can be seen on the Plan page of the Laserfiche Cloud Administration site, the Overview page of the Repository Administration site, and in your web browser address bar when viewing your Laserfiche documents.

  4. Leave the CreateCookie query option blank. It will default to “false”
  5. Within customerId, add your 9-digit Laserfiche Cloud account ID.
  6. Scroll down to the Request body section
  7. Type in a username and password. Note that this must be a real user account.
  8. Optionally, type in a value for the applicationName. All of the activity in the repository authenticated from this token will appear in your Audit logs with this information.
  9. Click Execute
  10. Locate the response body and copy the authToken value.
  11. Scroll to the top of the page, and click Authorize to display the following dialog box.
  12. Within Value, copy the authToken value copied from step 10.
  13. Click Authorize.
  14. This will automatically add the Authorization header to your requests with the bearer token.
  15. Now you're ready to make any other Laserfiche API call in the playground.

Note: The playground will automatically append the Bearer prefix to your authToken. When creating your integration, please make sure the Authorization header value has Bearer before your authToken, e.g., "Authorization: Bearer {authToken}"
