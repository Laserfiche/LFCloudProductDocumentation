Laserfiche API REST API Structure

Laserfiche API requests should take the form of:

{HTTP method} https://api.laserfiche.com/{service}/{version}/{resource}?{query parameters}

The components of a request include:

  • {HTTP method} — The HTTP method used for the request
  • {service} — The name of the Laserfiche service
  • {version} — The API version to use
  • {resource} — The path to the Laserfiche resource that you're referencing
  • {query parameters} — Optional OData query options or REST method parameters that customize the response

The response will include:

  • HTTP status code. The code will indicate the outcome of the request. See the Error Responses page for more details.
  • In the response body:
    • The requested data of the resource or the result of the request. See the Resource Object page for more details.
    • links — Requests that return large amounts of data may page their results. See the Paging page for more details.

HTTP Method

The HTTP method is also known as an HTTP verb.

Method Description Idempotent Semantic

GET Read data from the specified resource X Read

Search

POST Create a new resource, or perform an action Create

Bulk

PATCH Update a resource with new values Update

PUT Replace a resource with a new one X Replace

DELETE Remove a resource X Delete

GET and DELETE should not require a request body

POST, PATCH, and PUT may require a request body, typically in JSON API format

Service

Top-level services in the Laserfiche suite have an assigned abspath prefix. Service names are lowercase, even if they refer to a product name, and are usually singular. Some service names:

account

forms

repository

workflow

Refer to the API reference for the complete list of service names.
